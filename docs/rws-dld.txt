RealWorkStudio Website – DLD

(Full System – Frontend + Backend + Firebase + GitHub Integration)
0. GLOBAL SYSTEM ARCHITECTURE

Architecture Style

Frontend: Next.js (App Router, SSR + ISR)
Backend: Node.js (Express – Modular Monolith)
Database: PostgreSQL
Storage: Firebase Storage
External API: GitHub API
Auth (Admin): JWT-based
Deployment: Render
System Flow




User → Next.js (Frontend SSR)
     → Backend API (Express)
     → PostgreSQL (Data)
     → Firebase (Media)
     → GitHub API (Leaderboard)
Monorepo Structure (Recommended)




realworkstudio/
│
├── apps/
│   ├── web/          # Public Website (Next.js)
│   ├── admin/        # Admin Panel (Next.js)
│   └── api/          # Backend (Node.js)
│
├── packages/
│   ├── ui/           # Shared UI components
│   ├── config/       # Env, constants
│   └── utils/
Backend Architecture Pattern

Controller → Service → Repository → DB
DTO-based request/response
Central error handler
Middleware-based validation
Coding Standards

TypeScript everywhere
ESLint + Prettier
Strict typing (no any)
Environment-based config
Logging (Winston)
